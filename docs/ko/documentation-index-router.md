# 문서 컨벤션 인덱스 라우터

## 목적

`docs/en/index.md`는 문서 컨벤션 문서의 라우팅 진입점이다.
독자와 agent가 현재 작업에 필요한 문서 작성 규칙을 선택하도록 돕는다.
프로젝트 전체 문서 목록이 아니다.

## 기본 규칙

- 인덱스는 짧아야 한다.
- 인덱스는 `docs/` 아래 문서 컨벤션 문서만 라우팅해야 한다.
- 인덱스는 작업별 문서로 연결해야 한다.
- 인덱스는 각 문서를 언제 읽어야 하는지 설명해야 한다.
- 인덱스는 연결된 문서의 상세 규칙을 중복하면 안 된다.
- 인덱스는 다른 문서를 읽기 전에 적용되는 전역 문서 규칙을 포함할 수 있다.
- 인덱스는 모든 프로젝트 문서, 앱별 컨벤션 문서, 또는 연결된 문서의 세부 내용을 포함하면 안 된다.

## 읽기 경로

- 문서 링크는 엄격한 tree 구조일 필요가 없다.
- 필수 읽기 경로는 순환되지 않게 유지하는 것을 권장한다.
- `AGENTS.md`는 이 인덱스를 문서 컨벤션 진입점으로 가리키는 것을 권장한다.
- 이 인덱스는 작업별 문서로 바깥 방향 라우팅해야 한다.
- 작업별 문서는 사용되기 전에 독자가 이 인덱스로 돌아오도록 요구하면 안 된다.
- 관련 링크는 선택 참고 링크라면 순환될 수 있다.
- 전체 context를 위해 모든 관련 문서를 읽으라고 요구하면 안 된다.

## 필수 섹션

- `동기화 정책`: 언어 쌍 문서를 어떻게 일관되게 유지하는지 정의한다.
- `읽기 규칙`: 문서를 얼마나 읽어야 하는지 정의한다.
- `라우팅`: 작업 상황과 읽어야 할 문서를 나열한다.

## 비목표

인덱스는 다음 내용을 포함하면 안 된다.

- 전체 코딩 규칙
- 전체 아키텍처 설명
- 전체 도메인 정책
- 앱별 컨벤션 라우팅
- 긴 프로젝트 배경
- 연결된 문서에서 반복한 내용
- Linter, formatter, CI 규칙 목록

## 유지보수

- 새 문서 컨벤션 문서는 명확한 문서 작업 조건이 있을 때만 라우팅 섹션에 추가한다.
- 앱별 문서는 이 인덱스에 추가하지 않는다. 해당 앱의 문서 진입점에서 라우팅한다.
- 라우팅 설명은 짧게 유지한다.
- 인덱스에서 전체 문서를 요약하지 않는다.
- 필요한 경우 한글 번역 링크를 추가하거나 수정한다.
